Creating institution connections makes up the core of Meld's bank linking service and enables accessing customer financial data. Each connection has its own set of [products](https://docs.meld.io/docs/supported-bank-linking-products) associated with it, and is established via an underlying [service provider](https://docs.meld.io/docs/creating-connections#supported-products). Here are the steps to make a connection:

1. [Create a connect token.](https://docs.meld.io/docs/create-a-connect-token) Remember to choose the [products](https://docs.meld.io/docs/create-a-connect-token#supported-products) for which you would like data.
2. [Launch the widget.](https://docs.meld.io/docs/launch-the-widget)
3. [The user will select an institution.](https://docs.meld.io/docs/select-an-institution)
4. [The user will complete the connection.](https://docs.meld.io/docs/complete-the-connection)

# Routing

Routing can be used to route to user to the best provider to complete their connection. More information on bank linking routing can be found [here](https://docs.meld.io/docs/routing-overview#bank-linking-routing).


<Note>
Routing across Service Providers

Meld's Bank Linking stack allows you to route your customers to specific service providers based upon predefined defaults, a service provider waterfall based on institution coverage and availability, or specific financial institution based decisioning.

Routing rules are defined within the Meld dashboard.
</Note>
