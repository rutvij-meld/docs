title: "The Crypto Flows"
noindex: true
---

There are 4 main crypto flows that Meld supports, and this section will explain each of them in detail.

1. [Buy Flow](https://docs.meld.io/docs/buy-flow): Also known as onramping, this is when a user purchases crypto and sends it to a wallet address of their choice, in a single transaction.
2. [Sell Flow](https://docs.meld.io/docs/sell-flow): Also known as offramping, this is when a user sells crypto and gets paid for it via a payment method of their choice.
3. [Transfer Flow](https://docs.meld.io/docs/transfer-flow): This is when a user transfers crypto from a wallet they have on an exchange to another wallet address (custodial or non custodial) of their choice.
4. [NFT Checkout](https://docs.meld.io/docs/nft-checkout-flow): This is when a user purchases an NFT.

Note that all flows 4 essentially use the same endpoints. They all use the [widget endpoint](https://docs.meld.io/reference/crypto-session-widget-create), the get [transaction endpoint](https://docs.meld.io/reference/payments-transactions-get-1), and the buy and sell flow additionally use the [quote endpoint](https://docs.meld.io/reference/crypto-quote-get). Since they all use the same endpoints, the transaction type is determined by the params passed in.
