---
title: "ACH Transaction Statuses"
---

ACH transactions move through several statuses during their lifecycle. This page lists every status you might see when calling Meld's ACH endpoints and is intended for developers building or debugging an ACH integration. Statuses are either **TEMPORARY** (the transaction may still change) or **TERMINAL** (the transaction will not change again).

The most common lifecycle of a successful ACH transaction is `PENDING` → `SETTLED`. `PENDING` generally means the transaction has been submitted, and `SETTLED` means it has been batched by NACHA. All Meld ACH transactions are currently same-day ACH rather than standard ACH.

| Status     | Description                                                                                                      | State     |
| :--------- | :--------------------------------------------------------------------------------------------------------------- | :-------- |
| `PENDING`  | The transaction has been created in Meld's system and submitted to the service provider for review.              | TEMPORARY |
| `SETTLING` | The service provider has accepted the transaction but it has not yet been batched.                               | TEMPORARY |
| `SETTLED`  | The transaction has been batched and has completed successfully.                                                 | TERMINAL  |
| `DISPUTED` | The account holder whose money was debited has disputed the transaction and it is under review.                  | TEMPORARY |
| `FAILED`   | The transaction has been declined by external payment systems and/or cannot be processed.                        | TERMINAL  |

<Note>
For the full set of fiat transaction statuses across both ACH and card flows, see [Transaction Statuses](/docs/payments-stack/fiat-transaction-statuses).
</Note>
