---
title: "Creating a Customer"
---

The buyer (your customer) must always be represented by a unique Meld `customerId`. This page is for developers building an ACH integration who need to attach the buyer's name, email, and address to the customer record created during account verification. You (the merchant/seller) do not need to create a `customerId`.

In a Payin, you debit the buyer and credit your account. In a Payout, you credit the buyer and debit your account.

<Warning>
**Account verification is required before creating a customer for ACH transactions.**

Before beginning an ACH flow, you must verify the customer's bank account as described in [Making ACH Payments](/docs/payments-stack/fiat-payment-quickstart/making-ach-payments/index). Each customer you create must reuse the Meld `customerId` returned from the account verification flow.
</Warning>

<Note>
If the buyer is a repeat customer and you already know their Meld `customerId`, you can skip these steps.
</Note>

## Prerequisites

- A `customerId` returned by the Bank Linking account verification flow.
- The buyer's first name, last name, email address, and billing address.

## Steps

1. Call Meld's update customer endpoint (`PATCH /customers/{customerId}`) to update the existing `customerId` (from the bank linking connection) with at least the customer's first name, last name, and email address. If the bank account has several names on it, any one of the names is sufficient.
2. Call Meld's create customer address endpoint (`POST /customers/{customerId}/addresses`) to add an address to the customer. `lineOne`, `city`, `country`, `postalCode`, and `region` (state) are required.

An ACH customer must have a name, email address, and address. Once both calls succeed, you are ready to make an ACH transaction.

# API Calls

The Meld `customerId` is a path parameter for both of the following calls.

## Update a customer's name and email address

Sample request to `PATCH /customers/{customerId}`:

```json
{
    "name": {
        "firstName": "John",
        "lastName": "Smith"
    },
    "email": "testcustomer1@example.com"
}
```

Sample response:

```json
{
    "id": "WXDeHGgNJ1t1nFnoVY43Bh",
    "accountId": "W2bJdfJRoJQHjsNnUeUNRA",
    "externalId": "testcustomer1",
    "name": {
        "firstName": "John",
        "lastName": "Smith"
    },
    "email": "testcustomer1@example.com",
    "phone": null,
    "addresses": [],
    "serviceProviderCustomers": [],
    "status": "ACTIVE"
}
```

## Update a customer's address

Sample request to `POST /customers/{customerId}/addresses`:

```json
{
    "type": "BILLING",
    "address": {
        "firstName": "John",
        "lastName": "Smith",
        "lineOne": "123 Fake St",
        "city": "Boston",
        "region": "MA",
        "country": "US",
        "postalCode": "02112"
    }
}
```

Sample response:

```json
{
    "id": "WXDeHD8aBDD5c6dqBmNGFD",
    "customerId": "WXDeHGgNJ1t1nFnoVY43Bh",
    "type": "BILLING",
    "status": "ACTIVE",
    "addressDetails": {
        "lineOne": "123 Fake St",
        "lineTwo": null,
        "city": "Boston",
        "region": "MA",
        "country": "US",
        "postalCode": "02112"
    }
}
```
