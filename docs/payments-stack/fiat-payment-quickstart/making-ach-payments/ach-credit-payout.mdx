An ACH Payout is any transaction where money flows from the merchant to the buyer. This page is for developers implementing ACH credit. By the end you will have submitted a Payout and know how to fetch its final status.

## Prerequisites

<Info>
- A Meld `customerId` for the buyer — see [Creating a Customer](/docs/payments-stack/fiat-payment-quickstart/making-ach-payments/creating-a-payments-customer).
- A verified bank account on file for the buyer (account number, routing number, account type).
- An ACH-capable service provider configured on your account.
- Webhooks configured so you can receive `TRANSACTION_FIAT_CREATED` and `TRANSACTION_FIAT_UPDATED` events.
</Info>

## Steps

1. Call Meld's `POST /payments/transactions/payout` endpoint to make an ACH Payout. Pass in the buyer's `customerId` from the [Creating a Customer](/docs/payments-stack/fiat-payment-quickstart/making-ach-payments/creating-a-payments-customer) step. The request body carries the ACH details — `accountType`, `routingNumber`, and `accountNumber`.
2. Once the transaction is submitted, listen for the `TRANSACTION_FIAT_CREATED` webhook confirming the transaction was initiated. Use this webhook as a trigger to call `GET /payments/transactions/{transactionId}` to fetch the latest transaction details.
3. Depending on the underlying service provider, you may also receive a `TRANSACTION_FIAT_UPDATED` webhook telling you whether the transaction succeeded, failed, or moved to another status. Use this webhook as a trigger to refetch transaction details. For providers without webhook support, you will not receive updates after submission — but if the transaction fails at submit time you will see that in the response immediately.

<Note>
**Webhook support**

To see which providers support webhooks, refer to the [ACH Testing Guide](/docs/payments-stack/fiat-payment-testing-guide/ach-testing-guide).
</Note>

<Tip>
For ACH transaction status definitions and the typical lifecycle, see [ACH Transaction Statuses](/docs/payments-stack/fiat-payment-quickstart/making-ach-payments/ach-transaction-statuses).
</Tip>

# API Calls

## Make a Payout

Sample request to `POST /payments/transactions/payout`:

```json
{
    "customerId": "WXDeHGgNJ1t1nFnoVY43Bh",
    "sourceAmount": "15.00",
    "sourceCurrencyCode": "USD",
    "paymentMethod": {
        "type": "ACH",
        "details": {
            "accountType": "CHECKING",
            "routingNumber": "987654321",
            "accountNumber": "43214321"
        }
    },
    "serviceProvider": "MOOV"
}
```

`accountType` must be `CHECKING`, `SAVINGS`, or `BUSINESS_CHECKING`.
