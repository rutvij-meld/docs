Meld's Payments Stack supports several common fiat payment use cases. This page is a quick map for developers deciding which flow fits their integration before they jump into the API reference.

## Supported use cases

1. **Cards** — Meld supports charging users via card information through multiple providers. See the [Card Testing Guide](/docs/payments-stack/fiat-payment-testing-guide/card-testing-guide) for supported providers and test cards.
2. **ACH** — Meld supports ACH payin and payout transactions through multiple providers. See [Making ACH Payments](/docs/payments-stack/fiat-payment-quickstart/making-ach-payments/index) for the end-to-end flow.

<Note>
Before initiating an ACH transaction, you must [verify the user's bank account](/docs/bank-linking/bank-linking-quickstart/account-verification) using Meld's [Bank Linking Stack](/docs/bank-linking/bank-linking-quickstart/index). This lowers fraud and errors, and is the industry-standard methodology to get user-permissioned access to the account number and routing number (using the [IDENTIFIERS](/docs/bank-linking/bank-linking-quickstart/bank-linking-products/account-identifiers) product) and the account owner's name (using the [OWNERS](/docs/bank-linking/bank-linking-quickstart/bank-linking-products/account-owners) product).
</Note>
