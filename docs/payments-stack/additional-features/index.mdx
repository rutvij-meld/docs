These features let you further enhance or customize the payment transaction flow. This section is for developers who have a basic payment flow working and want to add capabilities like delayed capture or stored payment methods.

## In this section

- [Authorization & Capture](/docs/payments-stack/additional-features/authorization-capture) — split card payments into an authorization and a later capture for use cases like pre-orders.
- [Payment Processor Tokens](/docs/payments-stack/additional-features/payment-methods-and-tokenization) — tokenize cards with Meld so you can charge returning customers without re-collecting card data.
