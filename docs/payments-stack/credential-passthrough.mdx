---
title: "Credential Passthrough"
noindex: true
---

Credential passthrough lets you pass your customer's service provider credentials when sending a `POST` request, so a platform can process payments as one of the service provider's merchants. This page is for platform developers whose end-merchants hold direct relationships with the underlying provider.

<Note>
Refer to the API reference for sample code that uses these headers.
</Note>

# Service provider: Authorize.net

To pass through credentials when using Authorize.net, provide the following additional headers:

| Key                            | Type   | Description                                                       |
| :----------------------------- | :----- | :---------------------------------------------------------------- |
| `Meld-Passthrough-Key-Id`      | String | The unique identifier that the service provider has issued to you |
| `Meld-Passthrough-Key-Private` | String | The unique private key that the service provider has issued to you |
| `Meld-Passthrough-Reference`   | String | The unique reference code/ID that you assign to your customer     |
