---
title: "Error responses"
noindex: true
---

When a request fails, Meld endpoints return a response that includes an error code and message in the body. This page lists the error codes you might receive from Meld's fiat payment APIs and the action a developer should take in response. The error message includes both Meld's error code and the service provider's error details.

<Note>
For the shape of error responses, see the Error schema in the API reference.
</Note>

# General

| Error code      | Description                                                                                  | Recommended action                                                                                                                                                  |
| :-------------- | :------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `UNKNOWN`       | An unexpected error occurred.                                                                | Retry the request. If the error persists, contact Meld support with the request ID.                                                                                 |
| `BAD_REQUEST`   | The request body did not pass validation. The response includes a user-friendly description. | Inspect the response message (for example: `[amount] Must be a decimal value greater than zero`, `[serviceProvider] Must be a service provider`) and fix the request. |

# Payments

| Error code                                  | Description                                                                       | Recommended action                                                                                                                |
| :------------------------------------------ | :-------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------- |
| `INVALID_AMOUNT_OR_CURRENCY`                | The supplied amount or currency code is invalid.                                  | Send a positive decimal amount and a supported ISO 4217 currency code.                                                            |
| `INITIAL_TRANSACTION_ERROR`                 | Meld failed to create the initial transaction record.                             | Retry the request. If the error persists, contact Meld support with the request payload.                                          |
| `JSON_PROCESSING_ERROR`                     | The request body could not be parsed as JSON.                                     | Ensure the body is valid JSON and the `Content-Type` header is `application/json`.                                                |
| `PAYMENT_TRANSACTION_STATE_INVALID`         | The payment transaction is null or in an invalid state for the requested action.  | Refetch the transaction with `GET /payments/transactions/{transactionId}` and confirm its current status before retrying.         |
| `PREVIOUS_TRANSACTION_STATE_INVALID`        | A prior transaction referenced by this request is null or in an invalid state.    | Refetch the referenced transaction and ensure it is in a state that allows the requested action (for example, capture or refund). |
| `SERVICE_PROVIDER_ERROR`                    | The downstream service provider returned an error.                                | Inspect `serviceProviderDetails.response` for the underlying error and act per the provider's documentation.                       |
| `TRANSACTION_CREATION_ERROR`                | Meld was unable to create the transaction.                                        | Validate all required fields and retry. If the error persists, contact Meld support.                                               |
| `TRANSACTION_EXCEPTION`                     | An exception occurred while processing the transaction.                           | Retry the request. If the error persists, contact Meld support with the request ID.                                                |
| `TRANSACTION_INVALID_ID`                    | The transaction ID is invalid or was not found.                                   | Confirm the `transactionId` belongs to your account and was returned by a previous Meld response.                                  |
| `TRANSACTION_INVALID_STATE_ID`              | The transaction state ID is invalid or was not found.                             | Refetch the transaction and use the latest state ID.                                                                              |
| `TRANSACTION_NO_RESULT`                     | No result was returned from the gateway.                                          | Retry with the same `Idempotency-Key`. If the issue persists, contact the service provider and Meld support.                       |
| `TRANSACTION_NO_RESPONSE`                   | No response was received from the transaction submission.                         | Retry with the same `Idempotency-Key` to avoid duplicate charges. See [Idempotency Key](/docs/payments-stack/idempotency-key).      |
| `TRANSACTION_REQUEST_CREATION_ERROR`        | Meld was unable to build the request to send to the service provider.             | Validate all required fields, particularly `paymentMethod` and `serviceProvider`, and retry.                                       |
| `TRANSACTION_INVALID_PROVIDER`              | The specified service provider is invalid or not enabled on your account.         | Use a supported provider — see [Supported Service Providers](/docs/payments-stack/fiat-supported-service-providers).                |
| `TRANSACTION_UPDATE_NULL_ERROR`             | An attempt was made to update a null or invalid transaction.                      | Refetch the transaction before retrying the update.                                                                               |
| `TRANSACTION_INVALID_SERVICE_TRANSACTION_ID`| The `serviceTransactionId` is invalid.                                            | Ensure the value matches the ID returned by the service provider for the transaction.                                              |
| `TRANSACTION_NULL_DATA`                     | The transaction data was null.                                                    | Include all required fields in the request body and retry.                                                                        |
| `NOT_SUPPORTED`                             | The requested operation is not supported by the selected service provider.        | Check the provider's capabilities in [Supported Service Providers](/docs/payments-stack/fiat-supported-service-providers).          |
| `FAILED_GETTING_TRANSACTION`                | Meld was unable to fetch the transaction.                                         | Retry the GET request. If the error persists, contact Meld support with the transaction ID.                                        |
| `TRANSACTION_MISSING_SERVICE_PROVIDER_DETAILS` | The request requires a `serviceProviderDetails` property.                      | Include `serviceProviderDetails` as documented for the chosen service provider and retry.                                          |
| `TRANSACTION_MISSING_CARD_INFORMATION`      | The request is missing credit/debit card information.                             | Include a valid `paymentMethod` with card details, a token, or a nonce.                                                            |
| `TRANSACTION_FAILED_GETTING_CRYPTO_QUOTE`   | Meld failed to retrieve a crypto quote for the transaction.                       | Retry the request. If the error persists, contact Meld support.                                                                    |
| `TRANSACTION_CREDIT_CARD_INFO_NOT_SUPPORTED`| The provider does not accept raw card information for this flow.                  | Use a nonce or tokenized payment method instead — see [Payment Processor Tokens](/docs/payments-stack/additional-features/payment-methods-and-tokenization). |
