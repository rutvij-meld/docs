This section is for developers integrating any of Meld's three crypto products (White-Label API, Meld Checkout, or Virtual Account). The pages here cover the cross-cutting concepts — webhooks, transaction statuses, sandbox testing, and provider onboarding — you'll need regardless of which product you picked.

* **[Webhook events](/docs/stablecoins/for-all-products/webhook-events)** — the events Meld sends you when transactions are created and updated, with payload examples
* **[Webhook authentication](/docs/stablecoins/for-all-products/webhooks-authentication)** — how to verify webhook signatures
* **[Transaction statuses](/docs/stablecoins/for-all-products/transaction-statuses)** — the full list of crypto transaction statuses and what each one means
* **[Service-provider setup](/docs/stablecoins/for-all-products/service-provider-setup)** — how to add additional onramps for greater coverage and redundancy
* **[Sandbox guide](/docs/stablecoins/sandbox-guide)** — test cards, wallets, KYC triggers, and provider-specific sandbox notes
