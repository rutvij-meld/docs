---
title: "Unified KYC"
description: "This page shows you how to use Meld's Unified KYC product so that a user only has to KYC once"
---

Meld is unifying KYC, so that a user only has to KYC once and then can use multiple onramps without having to KYC again.

In this flow, the user can go straight from your wallet to the payment screen of the onramp, skipping both logging into the onramp as well as KYC.

**Currently this product is only supported for the KYC provider Sumsub and the onramp Mercuryo, however Meld will be adding support for more onramps (many of whom use Sumsub for KYC) as well as more KYC providers in the future.** As a result, this flow is currently only available for Retail Ramps, but will be made available in the future for all Meld crypto products as coverage increases.

There are 2 ways this flow can work:

1. You initiate KYC of the user. Here, you would have to integrate a KYC Provider, and make the requisite API calls to receive/launch their KYC link, track the KYC status of the user, and pass in the KYC token of the user to Meld. See more about this flow [here](https://docs.meld.io/docs/you-kyc-the-user).
2. Meld initiates KYC of the user. In this flow, you would call Meld's API to get back a KYC link and simply launch that from your UI. See more about this flow [here](https://docs.meld.io/docs/meld-kyces-the-user).

Meld strongly recommends flow #2 as it easier for you to integrate, keeps you out of the flow of sensitive data, and allows you to scale easily when Meld supports more KYC providers.
