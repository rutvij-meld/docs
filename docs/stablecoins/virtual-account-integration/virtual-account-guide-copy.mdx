# How White-Label API Virtual Account Integration Works

### High-Level Flow:

Since this flow can be used by individuals or businesses, the steps will say business to denote that.

1. **Manual** Meld will KYB the business with the onramp.
2. **Your UI:** Collect transaction preferences (amount, crypto, payment method)
3. **Meld API:** Get real-time quotes from multiple providers
4. **Your UI:** Display quotes and let business choose
5. **Meld API:** Create a Meld subAccount for your business (1 time per business per onramp)
6. **Meld API:** Create an order, and fetch the virtual bank account details or wallet address from the onramp (depending on if the business is buying or selling crypto)
7. For onramp: **Business' Bank App:** Business completes payment to the virtual bank account the onramp set up
8. For offramp: **Business' Wallet App:** Business sends crypto to the wallet address the onramp returned
9. **Webhooks:** Receive real-time transaction updates

### Technical Architecture:

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐   
│   Your UI   │───►│  Meld APIs  │───►│ Provider API│
│             │◄───│             │◄───│             │
├─────────────┤    ├─────────────┤    ├─────────────┤
│ • Quotes    │    │ • Pricing   │    │ • VA Info   │
│ • Selection │    │ • VA Info   │    │ • Status    │  
│ • Launch    │    │ • Webhooks  │    │ • Tracking  │
└─────────────┘    └─────────────┘    └─────────────┘
```

### Prerequisites:

* **API Key** from [Meld Dashboard](https://dashboard.meld.io)
* **Webhook endpoint** for transaction updates
* **Domain whitelisting for redirect URLs** (some providers require this)

# White-Label API Integration Steps

The virtual account flow will happen completely via your UI.

Here is a common integration approach using this three-step process:

## 1. Create Complete User Journey

Build your own custom UI to collect user preferences:

* Amount they want to spend/receive
* Fiat currency selection
* Cryptocurrency selection
* Bank transfer type preference (for example ACH vs Wire in the US)
* Wallet address for receiving crypto
* Country (can be auto-detected from business' device/browser instead of prompting)

## 2. Set Up Webhooks

Webhooks notify your server when transactions are created and updated. This step can be completed later.

### Requirements:

* **A publicly accessible URL** (localhost will not work)
* For testing, consider using [ngrok](https://ngrok.com/), [webhook.site](https://webhook.site/) or similar services

### Setup Process:

1. Navigate to **Developer > Webhooks** in the dashboard
2. Click **"Add Endpoint"**
3. Enter your **webhook URL** (must start with http/https)
4. Give your webhook a **descriptive name**
5. Select **"Subscribe to all events"**
6. Click **"Add endpoint"**


<Note>
### **Note:** You will receive notifications to this URL for all transaction creations and updates.
</Note>


## 3. Create Meld subAccount for the business

Call Meld's [Create SubAccount API](https://docs.meld.io/reference/accounts-sub-accounts-create) with the onramp's id for the business to create a meld subAccount for that business. You will receive the onramp's id for the user manually from Meld.

This only needs to be done 1 time per business per provider. If you it more than once, the Meld API will return an error indicating that the subAccount has already been created with Meld.

## 4. Fetch Quotes

Call Meld's [Quote API](https://docs.meld.io/reference/crypto-virtual-account-quote-get) to get real-time quotes from multiple onramp/offramp providers:

* Send user preferences to Meld quote API
* Receive quotes from available service providers
* Display quotes in your custom interface for business selection

### Key Response Fields Explained:

* **`sourceAmount`**: This is the amount of money the business will spend. For onramp this is in fiat, for offramp this is in terms of crypto.
* **`destinationAmount`**: This is the amount of money the business will receive. For onramp this is in crypto, for offramp this is in terms of fiat.
* **`totalFee`**: The total fee the business will pay, in terms of the fiat currency they are using.
* **`serviceProvider`**: The onramp whose quote you are viewing.
* **`rampScore`**, **`institutionName`**, **`lowKyc`**: These fields can be ignored as they are not relevant to the virtual account flow.

## 5. Initiate a Transaction

For the buy flow:

* Call Meld's [Create Buy Order API](https://docs.meld.io/reference/payments-virtual-account-ramp-onramp-order-create) with the business' wallet address and subAccountId to receive the virtual bank account details for the user.
* Have the business send money to the virtual bank account from their banking app.
* When the onramp receives the payment, they will send crypto to the business' wallet.

For the sell flow:

* Call Meld's [Create Sell Order API](https://docs.meld.io/reference/payments-virtual-account-ramp-offramp-order-create) with the business' wallet address, bank account information, and subAccountId to receive the wallet address for the user to send crypto to.
* Have the business send crypto from their wallet to the wallet address returned in the previous step.
* When the onramp receives the crypto, they will send payment to the business' bank account.

## 6. Track Transaction

Process webhooks and display transaction information to the business:

* Set up webhook endpoints to receive transaction status updates
* Fetch detailed transaction data from Meld's [Transactions API](https://docs.meld.io/reference/payments-transactions-get) when webhooks arrive
* Display transaction progress, completion status, and details in your UI

You will receive webhooks from Meld every time a transaction is created or updated. To learn more about the various Meld webhook events as well as see examples, click [here](https://docs.meld.io/docs/webhook-events).

#### Webhook Integration:

```javascript
// Sample Code: Webhook endpoint receives transaction updates
app.post('/webhook/meld', (req, res) => {
  const { transactionId, status, externalCustomerId } = req.body;
  
  // Use webhook as trigger to fetch full transaction details
  const transactionDetails = await fetchTransactionDetails(transactionId);
  
  // Update user's transaction in your database
  updateTransaction(transactionId, transactionDetails);
  
  // Notify user via websockets, push notifications, etc.
  notifyUser(externalCustomerId, {
    transactionId,
    status,
    message: getStatusMessage(status)
  });
  
  res.status(200).send('OK');
});

// Sample Code: Fetch transaction details when webhook arrives
async function fetchTransactionDetails(transactionId) {
  const response = await fetch(
    `/payments/transactions/${transactionId}`,
    {
      headers: {
        'Authorization': `BASIC ${apiKey}`,
        'Meld-Version': '2025-03-04'
      }
    }
  );
  return await response.json();
}
```

In the webhook, the `virtualAccountRampOrderId` value will match the `orderId` in the response of the previous step, allowing you to tie an order to it's corresponding webhook(s) and transaction.

1. **Extract the`paymentTransactionId`** from the webhook payload
2. **Call GET** [/payments/transactions/\{transactionId}](https://docs.meld.io/reference/payments-transactions-get)

### Transaction Status Display:

```javascript
// Sample Code: Display user-friendly status messages
function getStatusMessage(status) {
  const statusMessages = {
    'PENDING': 'Processing your transaction...',
    'SETTLING': 'Finalizing crypto transfer...',
    'SETTLED': 'Complete! Crypto sent to your wallet.',
    'FAILED': 'Transaction failed. Please try again.',
    'CANCELLED': 'Transaction was cancelled.'
  };
  
  return statusMessages[status] || 'Transaction status updated.';
}
```

## Best Practices & Tips

### Error Handling:

```javascript
async function handleAPIError(error, endpoint) {
  const errorMap = {
    401: 'Invalid API key or authentication failed',
    403: 'Access forbidden - check account permissions', 
    429: 'Rate limit exceeded - please wait and retry',
    500: 'Server error - please try again later'
  };
  
  const message = errorMap[error.status] || 'Unknown error occurred';
  
  // Log for debugging
  console.error(`API Error (${endpoint}):`, error);
  
  // Show user-friendly message
  showErrorToUser(message);
}
```

### Security Considerations:

* **Never expose API keys** in frontend code
* **Validate all user inputs** before API calls
* **Use HTTPS only** for API communications
* **Implement rate limiting** to prevent abuse
